# family_tree — Product Overview (Phase 1)

## Positioning
`family_tree` is a premium, cinematic, Hebrew-first web experience for a digital family legacy.
It is not a traditional genealogy chart; it is a living family universe built around people, relationships, and memories.

## Core promise
- Preserve **legacy** and **memory** with emotional depth.
- Make the family graph feel **alive** (motion, focus, atmosphere).
- Treat content as an **archive**: structured, filterable, time-aware.

## Main surfaces (MVP shape)
### Tree
- Interactive graph (zoom/pan/focus)
- Clickable person nodes
- Person side panel / popover
- “Alive-feeling” connections and transitions

### Feed
- Family content stream: photos, videos, posts, milestones
- Filters by person / branch / year / type
- Items link back to people and relationships

### Person profile
- Identity: names, dates, bio
- Relations summary
- Media gallery
- Timeline-ready structure

## Language & localization
- Hebrew-first UI (RTL)
- Architecture prepared for English later (fields for EN names; route structure ready for locale strategy)

## Phase 1 deliverable definition
Phase 1 is **foundation + product shell**:
- App Router foundation with a premium dark shell
- Tree/Feed route shells that feel real
- Mock data model that can map to a future backend
